# 00-general — Antigravity Workspace Rules (universal)

- Make changes in small, reviewable steps (1–3 files).
- Always write a short plan before editing.
- After editing: provide a minimal verification step (tests / run / manual check).
- Never introduce secrets or paste .env / tokens.
- If behavior changes: update docs/DEVLOG.md (3–6 bullets).
- Prefer simple, boring, maintainable solutions.
