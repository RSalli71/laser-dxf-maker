# Security Policy

## Supported Versions
Dieses Repository ist ein **Template**. Sicherheitsrelevante Anpassungen (z.B. Secrets-Handling, CI-Härtung) werden möglichst zeitnah umgesetzt.

## Reporting a Vulnerability
Bitte **keine** Sicherheitslücken als öffentliche GitHub Issues posten.

Stattdessen:
1. Schreibe eine E-Mail an: <SECURITY_EMAIL>
2. Beschreibe:
   - Was ist betroffen?
   - Wie kann man es reproduzieren?
   - Welche Auswirkung hat es?
3. Wenn möglich: Vorschlag für Fix/Mitigation

Wir melden uns so schnell wie möglich zurück.
