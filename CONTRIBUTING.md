# Contributing

Danke, dass du beitragen möchtest. Dieses Repo ist als **Template** gedacht – Beiträge sollen es **robuster, klarer und universeller** machen.

## Grundprinzipien
- **Kleine, isolierte Änderungen** (ein Thema pro PR)
- **Doku mitziehen** (README / docs / DEVLOG, wenn es hilft)
- **Keine Secrets** (Keys, Tokens, personenbezogene Daten)
- **Stack-neutral bleiben**: Template soll für viele Sprachen/Frameworks funktionieren

## Workflow
1. Issue anlegen (Bug/Feature-Template nutzen)
2. Branch erstellen (z.B. `fix/...` oder `feat/...`)
3. Änderungen umsetzen + lokal prüfen
4. PR erstellen (PR-Template ausfüllen)

## Qualität / Definition of Done
- Code/Dateien sind konsistent formatiert
- Änderungen sind nachvollziehbar (kurze Beschreibung im PR)
- Wenn sinnvoll: kurze Notiz in `docs/DEVLOG.md`
- Keine „toten“ Referenzen (Docs zeigen auf Dateien, die es auch gibt)

## Vibe/Agent Contributions
Wenn du Agents nutzt:
- Prompt/Plan im PR kurz dokumentieren (3–8 Zeilen reichen)
- Änderungen in **kleinen Iterationen** mit Zwischenchecks
- Bei größeren Entscheidungen: Eintrag in `docs/DECISIONS.md`

## Fragen / Diskussion
Wenn etwas nicht klar ist: Issue aufmachen und kurz beschreiben, was du erreichen willst.
